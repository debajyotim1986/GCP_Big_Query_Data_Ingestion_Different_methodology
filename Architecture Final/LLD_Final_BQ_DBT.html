System Architecture

Design Philosophy: Python owns orchestration, config loading, error branching, and audit writing. BigQuery owns all heavy computation — hash generation, delta joins, DQ SQL checks, and MERGE operations. This keeps the framework testable, versionable, and schema-agnostic.

Component Details

PipelineRunner
framework/pipeline_runner.py
Main orchestrator for a single table's pipeline. Called by Composer PythonOperator. Coordinates all other components in the correct sequence and handles top-level exception routing.
Input
registry_id, dag_run_id
Output
Run summary dict pushed to XCom
Raises
RuntimeError on DQ block or BQ failure
Cleanup
Drops staging table in finally block always
ConfigLoader
framework/config/config_loader.py
Reads table_registry and dq_rules from BQ at task runtime — always fresh, never cached. Returns typed TableConfig and DQRule dataclasses. Also supports bulk discovery of all active tables.
Input
registry_id or none for bulk load
Output
TableConfig dataclass with DQRule list
BQ reads
table_registry, dq_rules
Raises
ValueError if registry_id not found or inactive
ProcessControl
framework/config/process_control.py
Manages watermark timestamps and concurrency locking. Uses atomic BQ MERGE for lock acquisition — prevents two DAG runs from processing the same table simultaneously. Lock auto-expires after 4 hours.
Lock mechanism
Atomic MERGE on process_control table
Lock expiry
4 hours auto-release
Watermark
Only advanced on full success
First run
Returns None → triggers full load
DeltaDetector
framework/delta/delta_detector.py
Generates and executes a dynamic BQ SQL query that produces the staging table containing only changed or new records. Supports specific-column hash or full-row hash mode based on config.
Hash mode A
MD5 of specific columns only
Hash mode B
MD5 of TO_JSON_STRING(row)
Output
_delta_staging_{registry_id} table in BQ
Adds cols
_row_hash, _change_type (INSERT/UPDATE)
DQValidator
framework/dq/dq_validator.py
Executes all active DQ rules for a table against the staging table. Runs rules in priority order: REJECT first, then QUARANTINE, then WARN. Produces DQSummary with pass/fail counts per rule and handles row-level routing.
Rule types
NOT_NULL, RANGE, REGEX, UNIQUENESS, REFERENTIAL, CUSTOM_SQL
REJECT
Blocks entire pipeline, watermark not advanced
QUARANTINE
Bad rows removed from staging, logged
WARN
Logged only, pipeline continues
IngestionEngine
framework/ingestion/ingestion_engine.py
Generates a fully dynamic MERGE statement by introspecting target table schema from BQ metadata at runtime. Builds SET, INSERT column lists dynamically — zero hardcoded column names, works for any table.
Schema source
BQ table metadata introspection
Dynamic SQL
PK join, UPDATE SET, INSERT cols all generated
Returns
Count of rows inserted + updated
Excludes
_row_hash, _change_type from target write

framework/audit/audit_writer.py
Writes a complete run record to run_audit table at the end of every pipeline execution — regardless of success or failure. Captures all counts, timestamps, duration, and error messages.
Always runs

Streaming insert to run_audit table

SUCCESS, FAILED, DQ_BLOCKED, PARTIAL

Captures wall-clock time in seconds

Singleton wrapper around google-cloud-bigquery. Provides schema introspection, query execution, scalar queries, row queries, streaming inserts, and table management. All other components depend on this.

Delta Detection Flow

How hashing works: The framework generates an MD5 hash per row and compares it against the hash of the same row in the target table. If the hash differs or the row doesn't exist in target, it's a delta record. The hash can cover specific columns or the entire row.
Delta Detection — BQ SQL Execution Flow
flowchart TD INPUT([DeltaDetector.detect_and_stage called\nconfig + watermark_from + watermark_to]) INPUT --> META META[BQClient.get_table_meta\nIntrospect source table schema\nGet column list for hash building] META --> HASH_MODE HASH_MODE{delta_hash_cols\nconfigured?} HASH_MODE -- YES specific cols --> SPEC_HASH[Build specific-column hash\nCONCAT each col as STRING\nIFNULL handling for NULLs\nMD5 of concatenated string] HASH_MODE -- NO empty list --> FULL_HASH[Build full-row hash\nTO_JSON_STRING of entire row\nMD5 of JSON string] SPEC_HASH & FULL_HASH --> BUILD_SQL BUILD_SQL[Build staging SQL\nsource_snapshot CTE with hash\ntarget_hashes CTE with hash\ndelta CTE with LEFT JOIN + filter\nCREATE OR REPLACE TABLE staging] BUILD_SQL --> WM_FILTER WM_FILTER{watermark_from\nis NULL?} WM_FILTER -- YES first run --> NO_LOWER[WHERE watermark_col <= watermark_to\nNo lower bound - full table] WM_FILTER -- NO incremental --> WITH_LOWER[WHERE watermark_col > watermark_from\nAND watermark_col <= watermark_to] NO_LOWER & WITH_LOWER --> EXEC_SQL EXEC_SQL[Execute CREATE OR REPLACE TABLE\n_delta_staging_registry_id\nBQ Job runs hash join against target] EXEC_SQL --> COUNTS COUNTS[Count source rows in window\nCount rows in staging table\nReturn source_count and delta_count] COUNTS --> DONE2([Staging table ready\n_change_type = INSERT or UPDATE])
Generated SQL — Specific Column Hash Mode
-- Hash covers only: order_id, customer_id, amount, status CREATE OR REPLACE TABLE `project.framework_config._delta_staging_sales_orders_v1` AS ( WITH source_snapshot AS ( SELECT src.*, TO_HEX(MD5( IFNULL(CAST(src.`order_id` AS STRING), '__NULL__') || '||' || IFNULL(CAST(src.`customer_id` AS STRING), '__NULL__') || '||' || IFNULL(CAST(src.`amount` AS STRING), '__NULL__') || '||' || IFNULL(CAST(src.`status` AS STRING), '__NULL__') )) AS _row_hash FROM `source_project.raw.sales_orders` src WHERE src.`updated_at` > TIMESTAMP('2024-01-14T02:00:00') -- watermark_from AND src.`updated_at` <= TIMESTAMP('2024-01-15T02:00:00') -- watermark_to ), target_hashes AS ( SELECT T.`order_id`, T.`tenant_id`, TO_HEX(MD5(...same hash expr...)) AS _row_hash FROM `target_project.curated.sales_orders` T ), delta AS ( SELECT S.*, CASE WHEN TH.`order_id` IS NULL THEN 'INSERT' WHEN S._row_hash != TH._row_hash THEN 'UPDATE' END AS _change_type FROM source_snapshot S LEFT JOIN target_hashes TH ON T.`order_id` = S.`order_id` AND T.`tenant_id` = S.`tenant_id` WHERE TH.`order_id` IS NULL OR S._row_hash != TH._row_hash ) SELECT * FROM delta WHERE _change_type IS NOT NULL );

DQ Execution Flow

Rule execution order: REJECT rules run first — if any fail, the pipeline stops immediately without running QUARANTINE or WARN rules. This prevents unnecessary processing and gives the clearest signal on pipeline health.

DQ Rule Types — Deep Dive

NOT_NULL
REJECT recommended
Config params: none required
SELECT COUNT(*) AS failed_count FROM `staging_table` WHERE `target_column` IS NULL
Use for: Primary keys, mandatory foreign keys, required business fields. Typically set to REJECT — a NULL PK makes MERGE impossible.
RANGE
QUARANTINE recommended
Config params: {"min": 0, "max": 1000000, "inclusive": true}
SELECT COUNT(*) AS failed_count FROM `staging_table` WHERE `target_column` IS NOT NULL AND (`target_column` < 0 OR `target_column` > 1000000)
Use for: Numeric amounts, ages, quantities, percentages. QUARANTINE removes outliers without blocking the entire batch.
REGEX
QUARANTINE recommended
Config params: {"pattern": "^[A-Z]{2}[0-9]{6}$"}
SELECT COUNT(*) AS failed_count FROM `staging_table` WHERE `target_column` IS NOT NULL AND NOT REGEXP_CONTAINS( CAST(`target_column` AS STRING), r'^[A-Z]{2}[0-9]{6}$' )
Use for: Email formats, phone numbers, product codes, status enums, postal codes.
UNIQUENESS
WARN recommended
Config params: {"scope_cols": ["order_id", "tenant_id"]}
SELECT COUNT(*) AS failed_count FROM ( SELECT `order_id`, `tenant_id`, COUNT(*) AS cnt FROM `staging_table` GROUP BY `order_id`, `tenant_id` HAVING cnt > 1 )
Use for: Detecting duplicate records in a delta batch. Often caused by source CDC issues. WARN is safer as duplicates don't always indicate bad data.
REFERENTIAL
QUARANTINE recommended
Config params: {"ref_project":"p","ref_dataset":"d","ref_table":"customers","ref_column":"customer_id"}
SELECT COUNT(*) AS failed_count FROM `staging_table` S WHERE S.`customer_id` IS NOT NULL AND NOT EXISTS ( SELECT 1 FROM `p.d.customers` R WHERE R.`customer_id` = S.`customer_id` )
Use for: FK integrity checks across tables or datasets. Cross-dataset joins work natively in BQ.
CUSTOM_SQL
Any action
Config params: {"sql": "SELECT COUNT(*) FROM {table} WHERE amount < 0 AND status = 'DELIVERED'"}

Component: infra/bq/ddl/ — All 6 framework config tables + 3 data table templates. Run once per environment to bootstrap the framework. Apply via bq query, Terraform, or the setup script. 
-- ╔══════════════════════════════════════════════════════════╗
-- ║  table_registry                                          ║
-- ║  Purpose: Master config — one row per source→target pair ║
-- ║  Component: ConfigLoader reads this at every task start  ║
-- ╚══════════════════════════════════════════════════════════╝
CREATE TABLE IF NOT EXISTS `{project}.framework_config.table_registry`
(
  registry_id        STRING          NOT NULL,   -- Unique identifier e.g. 'sales_orders_v1'

  -- Source coordinates
  source_project     STRING          NOT NULL,
  source_dataset     STRING          NOT NULL,
  source_table       STRING          NOT NULL,

  -- Target coordinates
  target_project     STRING          NOT NULL,
  target_dataset     STRING          NOT NULL,
  target_table       STRING          NOT NULL,

  -- Delta detection config
  primary_key_cols   ARRAY<STRING>   NOT NULL,   -- e.g. ['order_id','tenant_id']
  delta_hash_cols    ARRAY<STRING>   NOT NULL,   -- empty array = hash all columns
  watermark_col      STRING          NOT NULL,   -- e.g. 'updated_at'

  -- Runtime options
  batch_size         INT64,                        -- NULL = no row limit
  is_active          BOOL            NOT NULL    DEFAULT TRUE,
  priority           INT64                         DEFAULT 100,  -- lower = runs first
  owner_team         STRING,                       -- for alerting routing
  tags               ARRAY<STRING>,               -- ['finance','critical']

  -- Audit
  created_at         TIMESTAMP       NOT NULL    DEFAULT CURRENT_TIMESTAMP(),
  updated_at         TIMESTAMP       NOT NULL    DEFAULT CURRENT_TIMESTAMP(),
  created_by         STRING
)
OPTIONS (
  description = 'Framework config: all registered source→target table pairs'
);
-- ╔══════════════════════════════════════════════════════════════════╗
-- ║  dq_rules                                                        ║
-- ║  Purpose: One row per DQ rule per table                          ║
-- ║  Component: DQValidator loads this per run — zero code to add    ║
-- ╚══════════════════════════════════════════════════════════════════╝
CREATE TABLE IF NOT EXISTS `{project}.framework_config.dq_rules`
(
  rule_id          STRING     NOT NULL,  -- e.g. 'dq_so_001'
  registry_id      STRING     NOT NULL,  -- FK → table_registry
  rule_name        STRING     NOT NULL,  -- Human-readable label

  rule_type        STRING     NOT NULL,
    -- Valid values: NOT_NULL | RANGE | REGEX | UNIQUENESS | REFERENTIAL | CUSTOM_SQL

  target_column    STRING,               -- NULL for multi-col or CUSTOM_SQL rules
  rule_params      JSON,                 -- {"min":0,"max":1000000} | {"pattern":"^..."} | {"sql":"..."}

  action_on_fail   STRING     NOT NULL,
    -- Valid values: REJECT | QUARANTINE | WARN

  rule_order       INT64                 DEFAULT 100,  -- lower = runs first within action group
  is_active        BOOL       NOT NULL  DEFAULT TRUE,
  description      STRING,               -- plain-English explanation of rule intent
  created_at       TIMESTAMP            DEFAULT CURRENT_TIMESTAMP()
)
OPTIONS (
  description = 'DQ rules per registered table — loaded at runtime by DQValidator'
);
-- ╔══════════════════════════════════════════════════════════════════════╗
-- ║  process_control                                                     ║
-- ║  Purpose: Watermark + concurrency lock state — one row per table     ║
-- ║  Component: ProcessControl reads/writes via atomic BQ MERGE          ║
-- ╚══════════════════════════════════════════════════════════════════════╝
CREATE TABLE IF NOT EXISTS `{project}.framework_config.process_control`
(
  registry_id           STRING     NOT NULL,  -- PK: one row per registered table

  -- Watermark state
  last_successful_run   TIMESTAMP,             -- NULL on first run; lower bound for next extraction
  last_run_id           STRING,                -- UUID of last SUCCESS run
  last_row_count        INT64,                 -- rows ingested in last success
  total_runs            INT64                  DEFAULT 0,

  -- Lock state
  status                STRING,                -- RUNNING | SUCCESS | FAILED | DQ_BLOCKED
  locked_at             TIMESTAMP,             -- NULL when unlocked
  locked_by             STRING,                -- DAG run_id holding the lock
  lock_expiry_hours     INT64                  DEFAULT 4,  -- auto-expiry threshold

  -- Audit timestamps
  first_run_at          TIMESTAMP,
  updated_at            TIMESTAMP
)
OPTIONS (
  description = 'Runtime watermark and concurrency lock per registered table'
);

-- ║  Purpose: Full history of every pipeline run (success + failure)     ║

-- ╚══════════════════════════════════════════════════════════════════════╝
CREATE TABLE IF NOT EXISTS `{project}.framework_config.run_audit`
(
  run_id               STRING     NOT NULL,   -- UUID generated per pipeline run
  registry_id          STRING     NOT NULL,   -- FK → table_registry
  dag_run_id           STRING,                  -- Airflow dag_run_id

  -- Extraction window
  watermark_from       TIMESTAMP,               -- NULL on first run
  watermark_to         TIMESTAMP  NOT NULL,

  -- Row counts
  source_row_count     INT64,
  delta_row_count      INT64,
  dq_passed_count      INT64,
  dq_failed_count      INT64,
  quarantined_count    INT64,
  ingested_insert_count INT64,
  ingested_update_count INT64,

  -- Outcome
  status               STRING     NOT NULL,   -- SUCCESS|FAILED|DQ_BLOCKED|PARTIAL|LOCK_CONFLICT
  error_message        STRING,                  -- populated on any failure

  completed_at         TIMESTAMP
)
PARTITION BY DATE(started_at)
CLUSTER BY registry_id, status
OPTIONS (

# ── Stage 1 — install dependencies only (no app source yet) ───────────── FROM python:3.11-slim AS builder ENV UV_NO_CACHE=1 \ UV_PROJECT_ENVIRONMENT=/app/.venv \ PYTHONDONTWRITEBYTECODE=1 WORKDIR /app # Install uv package manager RUN pip install uv --no-cache-dir # Copy lockfile + pyproject FIRST — Docker layer cache skips reinstall if unchanged COPY pyproject.toml uv.lock ./ # Install production deps only — no dev/test tools in image RUN uv sync --frozen --no-group dev --no-install-project
STAGE 2 — RUNTIME python:3.11-slim · copy venv from builder · non-root user · expose 8080
# ── Stage 2 — minimal runtime, no build tools ─────────────────────────── FROM python:3.11-slim AS runtime ENV PATH="/app/.venv/bin:$PATH" \ PYTHONUNBUFFERED=1 \ PYTHONDONTWRITEBYTECODE=1 \ PORT=8080 WORKDIR /app # Copy venv from builder — no pip or uv binary in final image COPY --from=builder /app/.venv /app/.venv # Copy application source only (framework + dashboard packages) COPY dashboard/ ./dashboard/ COPY framework/ ./framework/ # Run as non-root for Cloud Run security best practice RUN adduser --disabled-password --gecos "" appuser USER appuser # Cloud Run injects PORT env var — uvicorn binds to it EXPOSE 8080 CMD ["uvicorn", "dashboard.app:app", "--host", "0.0.0.0", "--port", "8080", "--workers", "2", "--log-level", "info"]
